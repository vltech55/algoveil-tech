Context

An enterprise health-tech client operated a high-volume inbound call center handling payer authorization requests. Each call required verifying member identity, retrieving policy coverage, capturing structured intake data, and routing to a human agent only for exceptions.

The existing IVR was rigid, average handle time was climbing, and the client was hiring rapidly to keep up. They wanted a voice AI that could handle the routine 80% of calls end-to-end with human backup on exceptions — without changing the caller's expectation of a fast, natural conversation.

The problem

Three constraints, all non-negotiable:

  • Round-trip latency under 500ms — anything slower reads as robotic and callers hang up.
  • Tenant isolation at the data plane, PHI-handling patterns that fail closed — sensitive records could never cross tenant boundaries, and the audit trail had to make that structurally provable.
  • Explicit handoff at exceptions — the system had to know when it wasn't confident and route to a human without silently guessing.
Latency was the make-or-break constraint. Everything downstream had to fit inside a strict budget, and every model call was a hard-priced decision. — From the engagement brief

Approach

Rather than chain three separate providers, we designed the pipeline as a single streaming graph where each stage could hand partial results forward before finishing:

  • STT streams token-by-token — the LLM starts generating on the first intent-carrying token, not after end-of-utterance.
  • LLM streams to TTS — TTS starts vocalizing on the first sentence-boundary token; the tail of the LLM response streams into the tail of speech synthesis without buffering.
  • Speculative endpointing — if the caller pauses mid-utterance, the LLM speculates an answer against the partial transcript and prewarms; if the caller keeps talking, the speculation is discarded.

All three model providers are wrapped behind a single tenant-scoped gateway that (a) enforces per-tenant model and rate policy, (b) writes every request/response to an append-only audit log keyed to tenant_id + call_id, and (c) never allows a request from tenant A to reach a session key or context belonging to tenant B — enforced at the database layer with Postgres row-level security fail-closed policies.

Latency budget

The 500ms round-trip target was split into a strict budget:

STT first-token → 80ms STT stream to LLM → 40ms LLM first-token (streaming) → 180ms LLM stream to TTS → 40ms TTS first-audio → 120ms Buffer / network → 40ms ───── Total 500ms

The LLM budget was the tightest and required (a) provider-side streaming enabled, (b) system prompt and few-shot examples kept under 2 KB, and (c) a single provider region colocated with the caller's WebRTC edge.

Tenant isolation

Sensitive records were designed to fail closed at the tenant boundary — a property the client's compliance team wanted structurally provable, not merely asserted at the application layer. The architecture supports that with three enforcement points:

  • Every table in the data plane has a tenant_id column enforced at insert with a database-level trigger.
  • Every read uses a session variable set from the authenticated JWT — Postgres RLS refuses any query that would return a row from a different tenant.
  • The gateway that fans out to LLM/STT/TTS uses per-tenant API keys where available; where a shared key must be used, request bodies are stripped of PHI and only opaque token IDs pass to the provider.

Operate & measure

Instrumented from commit one, not retrofitted at handoff. Prometheus scrapes per-stage latency histograms; Grafana dashboards track p50/p95/p99 by tenant, model, and endpoint. Every call is scored offline against a golden-set eval harness that runs on merge to main; regression above ε aborts the deploy.

Outcome

Cadence ran the target inbound queue end-to-end for the first month with a human agent monitoring exceptions. After the first quarter, average handle time dropped materially, the hiring plan was scaled back, and the operations team took over day-two ownership using the runbooks and Grafana dashboards handed off at close of engagement.

The system continues to run in production. The evaluation harness scores every deploy, and the audit trail on tenant isolation has held clean under review. The client's engineers now add golden-set examples as they discover them.
